## 二、关键缺陷与优化思路
### 2.1 架构层面缺陷
| 缺陷 | 影响 | 优化思路 |
|------|------|----------|
| **缺少 UPI 支持** | 无法适配用户已有基础设施 | 引入 InfrastructureMode + UserProvidedInfrastructure |
| **升级机制薄弱** | 升级风险高、无回滚 | 引入 ClusterVersion CRD + CVO 控制器 |
| **OS 适配硬编码** | 扩展成本高 | OSProvider 接口 + 注册机制 |
| **缺少 Asset 管理** | 无法追踪安装进度 | Asset 框架 + DAG 依赖图 |
| **节点配置不安全** | SSH 明文传输风险 | 支持 Ignition 声明式配置 |

## 五、品牌应用示例
### 5.1 代码命名
```go
// 扶摇基 - 蓝图管理器
package fuyaoblueprint

type BlueprintRegistry struct {
    assets map[string]Asset
    graph  *dag.DAG
}

type Asset interface {
    Name() string
    Dependencies() []Asset
    Generate(ctx context.Context, deps map[string]interface{}) (interface{}, error)
}

// 扶摇升 - 版本管理器
package fuyaoupgrade

type ClusterVersion struct {
    metav1.TypeMeta   `json:",inline"`
    metav1.ObjectMeta `json:"metadata,omitempty"`
    
    Spec   ClusterVersionSpec   `json:"spec,omitempty"`
    Status ClusterVersionStatus `json:"status,omitempty"`
}

type UpgradeOrchestrator struct {
    client.Client
    upgraders []ComponentUpgrader
}
```

```bash
# 扶摇基命令
fuyao blueprint init --config cluster.yaml

fuyao upgrade plan --version v1.29.0
fuyao upgrade apply --version v1.29.0
fuyao upgrade rollback --version v1.28.0
fuyao upgrade status
```
