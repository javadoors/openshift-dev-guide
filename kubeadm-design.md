```
部署方式	优点	缺点	适用场景

Kops	云原生、自动化	仅支持特定云平台	云上部署
```

## 1.4 术语定义
### 1.4.1 核心术语
| 术语 | 英文 | 定义 |
|------|------|------|
| **Bootstrap Token** | Bootstrap Token | 一种临时的、具有有限生命周期的令牌，用于新节点在加入集群时进行身份验证和获取证书。格式为 `[a-z0-9]{6}.[a-z0-9]{16}`。 |
| **TLS Bootstrap** | TLS Bootstrap | 一种自动化证书颁发机制，允许节点在首次加入集群时自动申请和获取客户端证书，无需手动分发证书。 |
| **CSR** | Certificate Signing Request | 证书签名请求，包含公钥和身份信息的请求对象，由节点生成并提交给集群 CA 进行签名。 |
| **CA** | Certificate Authority | 证书颁发机构，负责签发和管理数字证书的可信实体。在 Kubernetes 中通常指集群根 CA。 |
| **Static Pod** | Static Pod | 由 kubelet 直接管理的 Pod，其清单文件存放在特定目录中，无需通过 API Server 创建。 |
| **Control Plane** | Control Plane | Kubernetes 集群的控制平面，包含 API Server、Controller Manager、Scheduler 和 Etcd 等核心组件。 |
| **Phase** | Phase | Kubeadm 执行流程中的一个独立阶段，每个阶段完成特定的任务，如预检查、证书生成、配置上传等。 |
| **Kubeconfig** | Kubeconfig | Kubernetes 客户端配置文件，包含集群连接信息、用户凭证和上下文定义。 |

### 1.4.2 证书相关术语
| 术语 | 英文 | 定义 |
|------|------|------|
| **SAN** | Subject Alternative Name | 主题备用名称，证书中用于指定额外身份信息（如 DNS 名称、IP 地址）的扩展字段。 |
| **CA Certificate** | CA Certificate | 由证书颁发机构签发的根证书或中间证书，用于签发其他证书。 |
| **Server Certificate** | Server Certificate | 服务端证书，用于证明服务器身份并加密通信。 |
| **Client Certificate** | Client Certificate | 客户端证书，用于证明客户端身份。 |
| **Peer Certificate** | Peer Certificate | 对等证书，用于双向认证场景中双方的相互身份验证。 |
| **Certificate Key** | Certificate Key | 用于加密和解密证书数据的密钥，在多 Master 场景中用于安全传输证书。 |
| **Certificate Rotation** | Certificate Rotation | 证书轮换，自动更新即将过期证书的过程。 |

### 1.4.3 配置相关术语
| 术语 | 英文 | 定义 |
|------|------|------|
| **InitConfiguration** | InitConfiguration | Kubeadm 初始化配置，定义初始化集群时的参数，如 Bootstrap Token、节点注册信息等。 |
| **ClusterConfiguration** | ClusterConfiguration | 集群配置，定义集群级别的参数，如网络配置、版本信息、控制平面端点等。 |
| **JoinConfiguration** | JoinConfiguration | 加入配置，定义节点加入集群时的参数，如发现方式、控制平面配置等。 |
| **Discovery** | Discovery | 发现机制，节点加入集群时获取集群信息的方式，包括 Token Discovery 和 File Discovery。 |
| **Control Plane Endpoint** | Control Plane Endpoint | 控制平面端点，集群 API Server 的统一访问地址，在 HA 场景中通常指向负载均衡器。 |
### 1.4.4 组件相关术语
| 术语 | 英文 | 定义 |
|------|------|------|
| **kube-apiserver** | kube-apiserver | Kubernetes API Server，集群的统一入口，提供 RESTful API。 |
| **kube-controller-manager** | kube-controller-manager | 控制器管理器，运行集群控制循环的后台任务。 |
| **kube-scheduler** | kube-scheduler | 调度器，负责将 Pod 调度到合适的节点。 |
| **etcd** | etcd | 分布式键值存储，用于保存集群所有状态数据。 |
| **kubelet** | kubelet | 节点代理，负责管理节点上的容器生命周期。 |
| **kube-proxy** | kube-proxy | 网络代理，负责实现 Service 的网络规则。 |
| **CoreDNS** | CoreDNS | 集群内部 DNS 服务器，提供服务发现功能。 |
### 1.4.5 网络相关术语
| 术语 | 英文 | 定义 |
|------|------|------|
| **Service CIDR** | Service CIDR | Service 网络的 IP 地址范围，用于分配 ClusterIP。默认为 `10.96.0.0/12`。 |
| **Pod CIDR** | Pod CIDR | Pod 网络的 IP 地址范围，用于分配 Pod IP。由网络插件管理。 |
| **DNS Domain** | DNS Domain | 集群内部 DNS 域名后缀，默认为 `cluster.local`。 |
| **NodePort** | NodePort | 一种 Service 类型，通过每个节点的特定端口暴露服务。端口范围默认为 30000-32767。 |
### 1.4.6 高可用相关术语
| 术语 | 英文 | 定义 |
|------|------|------|
| **Stacked Etcd** | Stacked Etcd | 堆叠式 Etcd，Etcd 与控制平面组件运行在同一节点上的部署模式。 |
| **External Etcd** | External Etcd | 外部 Etcd，Etcd 集群独立于控制平面节点部署的模式。 |
| **HA** | High Availability | 高可用，通过冗余和故障转移机制确保服务持续可用。 |
| **VIP** | Virtual IP | 虚拟 IP，在 HA 场景中用于提供统一的访问入口。 |
| **Load Balancer** | Load Balancer | 负载均衡器，在多个 API Server 之间分发请求。 |
### 1.4.7 操作相关术语
| 术语 | 英文 | 定义 |
|------|------|------|
| **Init** | Initialize | 初始化，创建新的 Kubernetes 集群的过程。 |
| **Join** | Join | 加入，将新节点添加到已有集群的过程。 |
| **Upgrade** | Upgrade | 升级，将集群升级到新版本的过程。 |
| **Reset** | Reset | 重置，清理节点上的 Kubernetes 配置和数据，恢复到初始状态。 |
| **Drain** | Drain | 驱逐，安全地将节点上的 Pod 迁移到其他节点，通常在维护前执行。 |
| **Cordon** | Cordon | 封锁，将节点标记为不可调度，阻止新 Pod 调度到该节点。 |
| **Uncordon** | Uncordon | 解除封锁，恢复节点的调度能力。 |
### 1.4.8 安全相关术语
